# 2026-03-24 — Fresh project: cosmix-cosmic → cosmix

## Context

The `~/.gh/cosmix-cosmic/` workspace accumulated significant experimentation
around libcosmic, HTMX, sea-orm, candle embeddings, OpenClaw agents, and
COSMIC desktop integration. The Dioxus pivot (2026-03-21) made most of that
dead weight — the libcosmic UI crates (cosmix-calc, cosmix-view, cosmix-toot,
cosmix-mail), cosmix-web (HTMX+sea-orm), cosmix-embed (candle), and the
cosmix-daemon monolith (49KB main.rs touching Wayland, D-Bus, Lua, mesh, MCP,
agents) were all entangled with the old stack.

Rather than surgically removing libcosmic from a 11-crate workspace, started
fresh at `~/.gh/cosmix/` with only the code that aligns with the
implementation plan.

## What was carried over

| Crate | Rationale |
|-------|-----------|
| **cosmix-jmap** | The entire JMAP/SMTP server — 32 files, 4 SQL migrations, auth, filtering, SMTP inbound/outbound. Stack-aligned (axum + sqlx). No libcosmic dependency. |
| **cosmix-port** | ARexx IPC library + AMP wire format. 3 files, zero UI dependencies. Used by everything. |
| **db/schema.sql** | pgvector memory schema. Foundation for Phase 5 AI inference. |
| **_doc/** | 30 design documents from 2026-03-06 to 2026-03-18. Architectural context. |

## What was created fresh

| Crate | Purpose |
|-------|---------|
| **cosmix-mail** | Dioxus 0.7 desktop+web client. Feature-flagged: `default=["desktop"]`, `web`. WebKit black screen fix. Skeleton only — ready for Phase 6. |
| **cosmix-mesh** | Extracted mesh types (MeshNode, peer sync, WireGuard control) into a clean library crate. Scaffolded from daemon mesh/wg modules without the daemon entanglement. |

## What was dropped

- **cosmix-daemon** — 49KB main.rs monolith touching Wayland, D-Bus, Lua, mesh, MCP, agents. Too entangled. Mesh and agent functionality extracted into dedicated crates instead.
- **cosmix-calc, cosmix-view, cosmix-toot** — libcosmic desktop apps. Not part of the JMAP/mail/mesh plan.
- **cosmix-web** — HTMX + sea-orm web server. Plan uses axum + sqlx (already in cosmix-jmap).
- **cosmix-embed** — Candle-based embeddings. Plan uses Ollama API for embeddings.
- **cosmix-portd** — Generic Lua port daemon. Niche utility.
- **cosmix-markdown** — Depends on libcosmic. Will rewrite for Dioxus using pulldown-cmark (already in workspace deps).
- **All git submodules** — cosmic-comp, cosmic-settings-daemon, examine, observatory, clipboard-manager, wtype.
- **workspace/, scripts/, _bin/** — OpenClaw agent framework, deployment scripts, CLI wrappers tied to old structure.

## Result

Clean four-crate workspace:

```
cosmix/
├── Cargo.toml          # workspace: cosmix-jmap, cosmix-mail, cosmix-mesh, cosmix-port
├── crates/
│   ├── cosmix-jmap/    # JMAP+SMTP server (carried over, stack-aligned)
│   ├── cosmix-mail/    # Dioxus 0.7 client (new scaffold)
│   ├── cosmix-mesh/    # Mesh networking library (new, types from daemon)
│   └── cosmix-port/    # ARexx IPC + AMP (carried over)
├── db/schema.sql       # pgvector memory schema
├── _doc/               # 30 design docs
└── _journal/           # this file
```

CLAUDE.md written for the new workspace. Ready to develop against the
implementation plan without libcosmic baggage.
